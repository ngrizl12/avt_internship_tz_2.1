# Задание 1 - Анализ багов на странице карьерного сайта Авито
## Описание 
Проведен анализ багов на тестовой странице карьерного сайта Авито. В рамках задания были выявлены и задокументированы различные типы дефектов: функциональные, UI.
Все обнаруженные баги детально задокументированы в файле `BUGS.md` с указанием:
- Краткого описания дефекта
- Шагов для воспроизведения
- Ожидаемого и фактического поведения
- Серьезности бага
- Окружения тестирования

## Методология тестирования

Для сравнения с ожидаемым поведением использовался действующий продакшен-сайт [Авито Карьеры](https://career.avito.ru/). Это позволило определить, какие элементы и функциональности должны работать корректно на тестовой странице.

# Задание 2.1 - Проект автоматизированного тестирования API для сервиса объявлений Avito

## Описание проекта

Проект представляет собой систему автоматизированного тестирования REST API микросервиса объявлений. Реализованы тесты для всех основных эндпоинтов, покрывающие позитивные и негативные сценарии согласно тест-кейсам в файле `TESTCASES.md`.

#### Важно: Найденные в процессе тестирования баги задокументированы в файле `BUGS.md`. Тесты, выявляющие данные баги, помечены статусом `SKIPPED`, так как по условию задания все тесты должны быть пройдены.

## Структура проекта
```bash
api_autotests_tz_2.1/
├── tests/                         # Тестовые сценарии
│   ├── test_create_item.py        # Тесты создания объявлений
│   ├── test_get_item_by_id.py     # Тесты получения объявлений по ID
│   ├── test_get_items_by_seller.py # Тесты получения объявлений продавца
│   ├── test_get_statistics.py     # Тесты получения статистики
│   └── test_delete_item.py        # Тесты удаления объявлений по id
├── api.py                         # Основной клиент API
├── base_utils.py                  # Вспомогательные утилиты
├── validators.py                  # Валидаторы ответов API
├── requirements.txt               # Зависимости проекта
```
## API Эндпоинты
| Файл проекта | Тестируемый endpoint |
|------|---------|
| `test_create_item.py` |  POST /api/1/item - Создание объявления |
| `test_get_item_by_id.py` |  GET /api/1/item/{id} - Получение объявления по ID |
| `test_get_items_by_seller.py` |  GET /api/1/{sellerID}/item - Получение всех объявлений продавца |
| `test_get_statistics.py` |  GET /api/1/statistic/{id} - Получение статистики по объявлению |
| `test_delete_item.py` |  DELETE /api/2/item/{id} - Удаление объявления

## Стек

- **Python**
- **pytest** 
- **requests** 

## 1. Клонирование и настройка
```bash
# Клонировать репозиторий
git clone https://github.com/ngrizl12/avt_internship_tz.git
cd avt_internship_tz/api_autotests_tz_2.1

# Создать виртуальное окружение
python -m venv venv

# Установить зависимости
pip install -r requirements.txt
```

## 2. Запуск тестов
```bash
# Запустить все тесты
python -m pytest -v

# Запустить конкретный тестовый файл
python -m pytest tests/test_create_item.py -v
```
